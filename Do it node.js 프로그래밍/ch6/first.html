nodejs를 이용해서 클라우드 서비스를 이용할 시 관계형 데이터베이스는 비용이 더 많이 들거나 거의 대부분이 유료이다. 그래서 nosql의 DB를 사용하는 경우가 많다. 6장에서는 몽고디비를 활용하여 작업을 한다. 몽고디비는 불편하지만 성능면에서 뛰어나다. 대부분의 클라우드 서비스에서 지원을 해준다. 컬렉션(테이블비슷)->다큐먼트(한개의로우비슷)->그 안에 자바스크립트 객체가 들어갈 수 있다. 첫번째 : mongodb 설치 두번째 : path 경로 설정 세번째 : 사용자 경로에 /database/local 추가 네번째 : cmd 창 2개를 열고 하나는 mongod --dbpath C:\Users\lisha\database\local 명령 후 엔터 (몽고db서버라고 생각하기) 다섯번째 : 다른 cmd 창으로 들어가서 mongo -> use local -> db.users.insert({name:'코로나무섭다',age:25}) 분석 : db에 추가할건데~ users라는 콜렉션에(테이블) 추가할거야~ users는 만든적이 없지만 db자체에서 그냥 만들어버린다고 한다. 여섯번째 : 검색 -> db.users.find().pretty()를 하면 댐 -> 자동으로 _id가 생성 일곱번째 : 삭제 -> db.users.remove({name:/코로/}); -> name이라는 값이 코로가 들어가면 삭제를 하라는 의미 여덞번째 : db.users.insert({id:'test01',name:'림쨩',password:'1234'}) 다시 객체를 넣어줌 아홉번째 : robomongo 다운(UI기반 몽고디비 툴) , mongoose 다운 (관계형 데이터베이스 처럼 만들어줌,테이블을 정의하듯 스키마를 정의) 열번째 : 몽구스(connect->Schema->model) Schema는 테이블을 정의하는것과 비슷
